# Session: 2026-06-03

## Goal
Investigate a real "Out of Sync" / stuck-deletion on the ProPro folder surfaced by the app,
then add an in-app repair affordance for out-of-sync folders.

## Progress

### Diagnosis (real incident, drove the feature)
- App showed ProPro "Out of Sync" — 1 stuck deletion on `ClipSmart/02_Design`. Syncthing
  failed item: *"delete dir: directory has been deleted on a remote device but contains
  ignored files"*. Local dir held only a (gitignored) `.DS_Store`, so the delete stuck.
- **Root cause:** the ClipSmart `.git` was being synced over Syncthing. The icon work lived
  only on unmerged branch `feature/aloft-icons`; a HEAD flip to a `main`-based branch deleted
  those working-tree files, and Syncthing replicated the deletion to the other Mac.
- Confirmed Send & Receive folders have **no Revert/Override** (those are receive-only /
  send-only). The repair path is Rescan (+ the app's stuck-deletes Resolve for ignored-file deletes).

### Infra fixes (ProPro folder, both Macs) — see memory `project-syncthing-propro-setup`
- Recovered icons (`git checkout feature/aloft-icons -- 02_Design`); pushed 5 feature branches
  to GitHub; merged `feature/aloft-icons` → `main`; pushed; deleted branch local+remote; pruned stale worktree.
- Added `.git` + `.stversions` to `.stignore` on M1max **and** M4-Pro (code travels via GitHub, not Syncthing).
- Switched M1max versioning trashcan → **staggered** (maxAge 2851200s) via REST API to match M4-Pro.
- Verified stuck delete cleared: ProPro `idle`, need=0, 0 pull errors, global==local dirs (3315==3315).

### App feature — inline Rescan button
- `Views.swift` `FolderStatusRow.folderStatusLabel`: the "Out of sync" branch now renders a
  compact `arrow.clockwise` button (orange, `.help` tooltip) that calls `rescanFolder` — the
